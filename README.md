# Course Plan

## Typical UI Interview Process

### Stage 1
- **Frontend Interview** → Initial Screening

### Stage 2 — Screening
- **Phone Screen (UI Coding)** — *focus area*
- Phone Screen (Algo/DS)

### Stage 3 — Onsite
- System Design
- **JS Coding** — *focus area*
- **Component / UX Coding** — *focus area*
- Algo Coding
- Behavioral Interview

> Highlighted items (UI Coding, JS Coding, Component / UX Coding) are the primary focus of this course.

---

## Difficulty of the Questions

| Level | Description | Time | Notes |
|-------|-------------|------|-------|
| **Warm-up** | Very basic problems — solve quickly to get comfortable | 2–4 minutes | — |
| **Easy** | Small, focused problems | 5–10 minutes | Often 3–4 questions in screening interviews |
| **Medium** | Core interview difficulty | 15–20 minutes | Most frontend interview questions live here |
| **Hard** | Complex problems requiring practice — browser APIs, edge cases, real-world constraints | 45–60 minutes | Workshop target: 20–25 minutes |
| **EXTREME!** | End-to-end product-style problems — build a feature from scratch; architecture & structure matter most | 1–2 hours | Typical for **Staff-level** interviews |
